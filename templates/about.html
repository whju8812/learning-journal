About this project

我的 AI 驅動
軟體學習日誌

這不是一個新聞閱讀器。這是我的私人學習筆記——只不過筆記是由 Claude 每天自動撰寫的。

這是什麼

一本不用自己寫的學習日誌

軟體世界每天都在發生新的事。但我沒有時間消化 Hacker News、GitHub、各種技術 blog 的每一條訊息——更難的是， 即使看了，也不一定知道「這件事跟我的學習方向有什麼關係？」

這個日誌解決的問題很具體：每天早上，自動告訴我「今天技術界發生了什麼，對我的 5 個學習方向各自意味著什麼」。

不是 RSS 摘要，不是 AI 產生的廢話。是 Claude 真的做了研究、真的看過今天的討論之後， 用我懂的語言、按照我在乎的分類，寫給我的一篇筆記。

最重要的設計決策： Claude 是作者，不是過濾器。 每篇日誌是 Claude 主動研究、綜合、分析之後的輸出——不是把標題重新排列一遍。
如何運作

完全自動化的流程

整個系統每天早上在無人介入的情況下自動完成。

1
Claude Code 每天執行學習腳本
每天 08:00 與 20:00（台灣時間），Claude Code CLI 自動執行預設 prompt，啟動一個學習 session——不需要手動觸發任何事情。
Claude Code · 每日兩次
2
Claude 搜尋今日技術動態
Claude 使用 web search 主動查詢 Hacker News、GitHub Trending、AI/ML 新發布、Threads 技術帳號——等同於把今天該看的東西都看過一遍。
web_search tool
3
Claude 撰寫並產生 GitHub Actions Workflow
把研究結果整理成結構化日誌（技術摘要 + 應用場景 + 5 方向分析），然後產生一個一次性的 GitHub Actions workflow 檔案並 commit push 到 repo。
GitHub Actions Workflow
4
GitHub Actions Runner 執行 POST
Workflow 觸發後，GitHub 的 ubuntu-latest runner 使用企業 IP 呼叫 POST /api/entries——繞過 Vercel 對雲端 IP 的封鎖限制。
ubuntu-latest runner · POST /api/entries
5
Flask API 驗證並儲存
網站後端驗證 API key，檢查資料格式，存入 Supabase（PostgreSQL）。全程無 AI 在伺服器端——只是標準的 REST API 操作。
Flask · Supabase
6
我打開網站，閱讀今天的筆記
日誌頁面顯示日期側欄，可以點選任何一天的紀錄，切換「技術內容」、「應用場景」和「學習方向分析」分頁。
index.html · Vanilla JS

為什麼用 GitHub Actions 而不直接 curl？ Vercel 會封鎖來自雲端 IP（包含 Claude Code 執行環境）的直接 HTTP 請求。 GitHub Actions runner 使用已知企業 IP，不受此限制——所以讓 runner 代為 POST。
