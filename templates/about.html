About this project

我的 AI 驅動
軟體學習日誌

這不是一個新聞閱讀器。這是我的私人學習筆記——只不過筆記是由 Claude 每天自動撰寫的，一天四次。

這是什麼

一本不用自己寫的學習日誌

軟體世界每天都在發生新的事。但我沒有時間消化 Hacker News、GitHub、各種技術 blog 的每一條訊息——更難的是， 即使看了，也不一定知道「這件事跟我的學習方向有什麼關係？」

這個日誌解決的問題很具體：每天台灣時間 01:00、04:00、07:00、10:00，自動告訴我「這幾小時技術界發生了什麼，對我的 5 個學習方向各自意味著什麼」。

不是 RSS 摘要，不是 AI 產生的廢話。是 Claude 真的做了研究、真的看過今天的討論之後， 用我懂的語言、按照我在乎的分類，寫給我的一篇筆記。

最重要的設計決策： Claude 是作者，不是過濾器。 每篇日誌是 Claude 主動研究、綜合、分析之後的輸出——不是把標題重新排列一遍。
如何運作

完全自動化的流程

整個系統每天四次、在無人介入的情況下自動完成。

1
遠端排程觸發器喚醒 Claude Code
台灣時間 01:00、04:00、07:00、10:00，Claude Agent SDK 的排程觸發器自動啟動一個 Claude Code CLI session，不需要我手動介入任何事情。
Claude Agent SDK · Remote Trigger
2
Claude 搜尋最新技術動態
Claude 使用 web search 工具主動查詢最新的技術新聞、社群討論、GitHub 趨勢，並比對當天已有的 session 內容，避免重複主題。
web_search tool
3
Claude 建立一次性 GitHub Actions workflow
把研究結果整理成結構化日誌（技術摘要 + 5 方向分析），寫入一個一次性的 .github/workflows/write-journal-entry-{日期}-{時間}.yml 檔案，commit 並 push 到 GitHub。
GitHub · git push
4
GitHub Actions runner 呼叫日誌 API
GitHub runner 讀取 workflow，以 POST 請求把 JSON 日誌寫入網站 API。Vercel 會封鎖雲端 IP 的直接請求，所以 Claude 不直接呼叫——GitHub Actions 才是合法的寫入路徑。
POST /api/entries
5
Flask API 驗證並儲存
網站後端驗證 API key，檢查資料格式，存入 Supabase（PostgreSQL）。全程無 AI 在伺服器端——只是標準的 REST API 操作。
Flask · Supabase
6
我打開網站，閱讀筆記
日誌頁面顯示日期側欄，可以點選任何一天的紀錄，以 session 時間切換，查看「技術內容」和「學習方向分析」兩個 tab。
index.html · Vanilla JS

為什麼要繞過 GitHub Actions 才能寫入？ Vercel 封鎖來自雲端 IP 的直接 HTTP 請求，Claude Code 的執行環境就在其中。 所以 Claude 不直接 POST API——而是建立一個一次性 GitHub Actions workflow 推送到 repo， 由 GitHub runner（使用合法企業 IP）代為呼叫 API 完成寫入。
